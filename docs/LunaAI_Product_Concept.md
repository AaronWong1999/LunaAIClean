# LunaAI: 极速新闻驱动的 Polymarket 交易特工

## 一、 一句话讲清楚 LunaAI（Elevator Pitch）

> **完整版：**
> LunaAI 是一款类似于 Bloomberg Terminal 的 Web3 预测市场极速交易 Agent；它能秒级捕获全球突发新闻，利用大模型瞬间映射到对应的 Polymarket 交易标的，并通过交叉验证新闻爆发前“聪明钱”的内幕建仓动作，为你提供从“一键跟单”到“纪律性止盈退场”的全自动交易闭环。
> 
> **极简/电梯版：**
> 市面上唯一由**「突发新闻 + 内幕资金」双重驱动**的 Polymarket 全自动交易特工，解决你“看新闻慢半拍、跟单抓瞎、会买不会卖”的痛点。

## 二、 核心差异化（为什么你和别人不一样？）

市面上的 TG 机器人大多只是“单点工具”，要么单纯推新闻，要么单纯监控大户地址，用户从接收到信息到最后完成获利用时极长。LunaAI 把这些割裂的环节做成了**共振式闭环**：

1. **从“资讯消费”到“瞬间变现”**
   - **痛点**：推特上刷到突发事件，再去 Polymarket 搜索相关池子（比如“特朗普遭遇刺杀”找总统大选赢家），往往发现赔率早就被抹平了。
   - **你的差异**：新闻发布毫秒级抓取 -> LLM 瞬间吃透语义并定性（利多/利空） -> **向量检索直接匹配到对应的 Polymarket 合约 Slug**，用户收到推送时，底下已经附带着“一键做多/做空”的按钮。
2. **不仅仅是抓聪明钱，而是抓“内幕钱”的交叉验证**
   - **痛点**：常见的 Smart Money 警报满天飞，胜率并不稳定，很多大户也在盲猜。
   - **你的差异**：LunaAI 只有在**新闻触发**时，回头去扒该标的在**新闻发生前 1~2 小时内**是否有巨额异常买入。这种提前布局的绝大多数是知情者/内幕交易，用突发新闻去倒推出来的聪明钱，含金量与跟单胜率是极高的！
3. **“授人以渔”的纪律性出场策略（极具价值）**
   - **痛点**：Polymarket 玩家最大的问题是“会买不会卖”。由于所有的期权最终会走向 $0 或 $1，很多原本浮盈几倍的单子最后归零。
   - **你的差异**：一键介入后，Bot 直接接管退场。你可以选择：
     - **“翻倍出本”策略**：到了 2x 均价，智能合约自动抛售一半，剩下利润狂奔。
     - **“跟庄撤退”策略**：监听带你进场的那个“内幕聪明钱”，他在链上点下了 Sell，Bot 立刻在 1 秒内帮你 Sell。
     - **“固定盈亏比”策略**：直接挂类似于传统金融的限价止盈/保护性止损单。

## 三、 把这事做到极致：可行性与难点拆解

这套产品逻辑非常犀利（极度贴近 Web3 的黑暗丛林属性），要在工程上做到极致，是完全有可能的，这里是技术难点及解法拆解：

### 1. 极速新闻捕获 (The Bloomberg Mechanism)
要追求极致速度，普通的免费抓取太慢。
- **解法方案**：建立高信噪比的新闻池。使用推特监控底层 API（如直接挂载类似 Tier10k, unusal_whales, 顶级新闻源的发推 Webhook），或者接入 Tree News (非常多 Crypto Degen 交易员使用的极速新闻源)。速度就是一切，省去 5 秒钟可能就意味着赔率从 20% 变到 50%。

### 2. LLM 语义与标的瞬时映射 (The LLM Router)
几千个 Polymarket 池子，新闻来时怎么知道应该买哪个？
- **解法方案**：使用 RAG (检索增强生成) 结合轻量级高速大模型（如 Groq 驱动的 Llama3，延迟几百毫秒）。
  - **日常动作**：每天爬取、清洗 Polymarket 所有活跃市场 (Active Markets)，建立本地向量数据库 (Vector DB)。
  - **触发动作**：新闻到来 -> 提取关键词生成 Embedding -> 向量检索出 Top 3 相关池子 -> 传入大模型判断该新闻对池子是 "Long/Yes" 还是 "Short/No"。

### 3. “内幕聪明钱”前置查询 (The Insider Tracker)
这一步是该 Bot 的“灵魂”。
- **解法方案**：由于 Polymarket 在 Polygon 链上交易（高并发、低 Gas），你需要通过 RPC 节点或专门的链上数据索引（The Graph 或自行抓取 Polymarket CTF 合约事件）。如果在匹配到池子的大约同一秒，通过 SQL/脚本逆查过去 1 小时该 TokenID 的大额 `TransferSingle` 买盘地址，就能过滤出潜伏名单。这部分工程量稍大，但完全可实现。

### 4. 自动化交易与退出执行 (Programmatic Execution)
- **解法方案**：需要用户提供专用钱包私钥去签名交易，或者使用较安全的智能合约钱包代理（需考虑安全性设计）。
  - 挂单方式可以使用 Polymarket 的限价单 CLOB API（链下撮合），对 Gas 和滑点更友好。监控聪明钱卖出可以通过监听特定地址的订单广播来实现。

## 四、 总结与 MVP 建议

这是极具战斗力的产品定位，一旦跑通，对于预测市场的 Degen 来说是**降维打击工具**。

建议你的第一期 MVP 这么干：
1. 先不搞庞杂的新闻源，**只盯紧 10 个特定的“爆料体质”推特账号**。
2. 标的映射先做**最头部的政治/重大时事池**，不做长尾杂项。
3. 退出策略先只做最无脑也是最刚需的：**“翻倍出本”和“定损（比如亏30%就跑）”**。

这套逻辑一旦完成，你可以自己拿资金跑，等赚到钱产生实际案例记录截图后，再开放 VIP 会员费（而不是免费开放，这个策略太卷会把 Alpha 磨没）。
